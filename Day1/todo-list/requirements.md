# ToDoアプリ 要件定義書（改良版）

## 1. アプリ概要

### アプリ名
ToDoアプリ

### 目的
Webブラウザ上で、  
**1つのタスクごとに「ToDo管理・タイマー・メモ」をまとめて扱える**  
シンプルで学習向けのタスク管理アプリを提供する。

### コンセプト
- 1タスク = 1まとまり（完了状態 / タイマー / メモ）
- 機能は最低限、構造は分かりやすく
- JavaScript初学者が「読んで理解できる」設計

### ターゲットユーザー
- 簡単なリスト管理をしたい一般ユーザー
- HTML / CSS / JavaScript 学習中の初心者

---

## 2. 機能要件

### 2.1 タスク管理機能（ToDo）

#### 機能一覧
- タスクの追加
- タスクの削除
- タスク名の編集
- タスクの完了チェック

#### 詳細仕様
- タスクはテキスト入力で追加する
- 各タスクにはチェックボックスを表示する
- 完了したタスクは以下のように表示を変更する
  - 文字色を薄くする
  - または打ち消し線を表示する
- 編集時は入力欄に切り替える簡易的な方式とする

---

### 2.2 タスク別タイマー機能

#### 機能一覧
- タイマー開始
- タイマー停止
- タイマーリセット
- 経過時間の表示

#### 詳細仕様
- タイマーはタスクごとに1つ持つ
- 表示形式は「分：秒（mm:ss）」とする
- 同時に動かせるタイマーは1つまで
  - 別のタスクのタイマーを開始すると、前のタイマーは停止する
- タイマーの状態は画面上に即座に反映される

---

### 2.3 タスク別メモ機能

#### 機能一覧
- メモ入力
- メモ内容の保存
- メモ内容の復元

#### 詳細仕様
- 各タスクにテキストエリアを配置する
- メモは入力と同時、または入力完了時に保存する
- 保存先は localStorage とする
- ページを再読み込みしてもメモ内容は保持される

---

### 2.4 データ保存機能（簡易）

#### 保存対象
- タスクの完了状態
- タスクのメモ内容

#### 保存方法
- localStorage を使用
- タスクIDをキーとして保存する
- タイマーの経過時間は保存しない（簡略化）

---

## 3. 非機能要件

### パフォーマンス
- ボタン操作後、**即座に画面が更新される**こと
- ページ全体の再読み込みは行わない

### 使いやすさ
- 操作はすべて1画面で完結する
- ボタン・チェックボックスの役割が一目で分かる
- 初心者ユーザーでも直感的に操作できる

### 開発・保守性
- JavaScriptのコードは短く、読みやすく書く
- 処理は「タスク単位」でまとめる
- 変数名・関数名は意味が分かる名前を使う
- 複雑な設計パターンは使わない

---

## 4. 画面構成（UIイメージ）

| ToDoアプリ |
| [タスク入力欄] [追加] |
| □ タスク名 [編集] [削除] |
| タイマー：00:00 |
| [開始] [停止] [リセット] |
| メモ： |
| [テキストエリア] |
| ☑ タスク名（完了） |
| タイマー：05:30 |
| [開始] [停止] [リセット] |
| メモ： |
| [テキストエリア] |

- 縦にタスクを並べる
- 各タスクはカードのように区切る

---

## 5. 扱うデータ

### タスクデータ構造（例）

- id（数値または文字列）
- title（タスク名）
- isCompleted（完了状態）
- memo（メモ内容）
- time（タイマー経過秒数）

### 管理方法
- JavaScriptの配列でタスク一覧を管理
- localStorage は補助的に使用する

---

## 6. 前提条件・制約

- 使用技術
  - HTML
  - CSS
  - JavaScript
- フレームワーク・外部ライブラリは使用しない
- サーバー通信は行わない
- Webブラウザ（PC）での利用を前提とする
- 学習用のため、仕様追加は最小限に留める

---

## 7. スコープ外（今回やらないこと）

- ユーザー認証・ログイン機能
- データベース連携
- 複数端末でのデータ同期
- タイマーの複数同時実行
- デザインの高度なアニメーション
- スマホ専用UI対応
